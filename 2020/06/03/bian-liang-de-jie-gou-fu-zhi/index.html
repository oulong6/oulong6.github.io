变量的解构赋值

js
javascript
发布日期:   2020-06-03

变量的解构赋值

数组的解构赋值


let [x,y] = [1,2];
//同等于
let x = [1,2][0];
let y = [1,2][1];

数组的解构赋值是按照元素的顺序位置对变量进行赋值的


let [x,y,z] = [1,2]
//解构不成功

当解构不成功那么变量的默认值是undefined

解构不完全


let [x,y] = [1,2,3]
//解构不完全

解构不完全并不会有错误

被解构的对象

被解构的对象一定要是数组,如果不是数组那么要有iterator接口,如果两个条件都不满足,那么将会报错

数组解构可以被嵌套


let [[x]] = [[1]]
//x = 1;

这就是嵌套数组解构

可以给变量设置默认值

当解构后变量的默认值如果等于undefined那么,将会使用变量的默认值

对象的解构赋值


let {name} = {name: 'Tom'}
//同等于
let obj = {name: 'Tom'}
let name = obj.name

对象的解构赋值是按照对象的属性名匹配模式进行解构赋值的,也就是变量名要与被解构的对象属性名一致
才能正确地进行对变量的赋值

解构不成功

如果解构不成功那么变量的值是undefined,并且如果变量有默认值,那么如果解构后变量的值是undefined,那么将会用变量默认值对变量进行赋值操作

二次解构

对解构后的变量再次解构


let {info: {name}} = {info: {name: 'Tom'}}
//
//此时的name == Tom

其实这里info并不是变量,它是匹配模式,匹配解构对象中同名的属性


let {name} = {name: 'Tom'}
//其实它是下面的简写;当匹配名与变量名相同是可以简写
let {name: name} = {name: 'Tom'}

被解构的值不能是undefined 和 null

因为undefined和null不能转换成对象

被解构的对象可以是基本类型(string,number,boolean,symbol)

因为当被解构的对象是基本类型时,会将它转成对象
但是undefined与null不能转换成对象,所以不能是undefined与null

并且也可以指定默认值,当结构后变量的值是undefined,使用默认值对变量进行赋值

变量解构不能用圆括号,变量赋值可以有圆括号


let [(x)] = [1] //错
let ([y]) = [1] //错
let {(x)} = {x: 12} //错

let x ;
({x}= {x: 12}) //对
