对变量|函数提升的理解

变量函数提升


function f(){
 console.log(foo) //undefined
 console.log(typeof x)   //function
 var foo = 12
 function x() {

 }
}

这就是 变量|函数 提升,怎么理解呢
当js脚本执行时,如果是在浏览器中,那么会创建一个全局执行上下文,执行上下文的生命周期为两个阶段,创建阶段与执行阶段,在创建阶段生成变量对象,确认作用域链,确定this的指向;
创建变量对象,变量对象的属性是在当前执行上下文中声明的变量与声明的函数;变量属性的初始值是undefined ,而函数的初始值就是指向函数对象的引用地址;
变量对象的属性是这样创建的

  1. 有变量声明,判断变量对象是否有与这个标识符同名的属性,如果没有则为变量对象创建于标识符同名的属性,并且初始值为 undefined;如果变量对象有与这个标识符同名的属性,那么忽略这个变量声明;
  2. 有函数声明,判断变量对像是否有与这个标识符同名的属性,如果没有则为变量对象创建与标识符同名的属性并且属性值为函数对象的引用值,如果变量对象已经有同名的属性则将属性值更改为新的函数对象的引用值;

在代码开始执行时,上下文为执行阶段;此时变量对象以创建;所以使用变量,并不会报错,而是返回变量对象中的值;
