谈谈html与css的规范

1.在一些书上会看到

<!DOCTYPE html>

指定HTML的文档类型的HTML5,并且解析模式为标准模式,相对的是混杂模式与接近标准模式,标准模式与非标准模式的解析方式不同,并且css与js的执行有一些的不同,
混杂模式是向后兼容的;

<!doctype html>

不信你看B站的页面
2.在想要对IE做一些特殊的兼容处理时,带上下面这个标签

<meta http-equiv="x-ua-compatible" content="IE=edge">

IE 支持通过特定的 标签来确定绘制当前页面所应该采用的 IE 版本。除非有强烈的特殊需求，否则最好是设置为 edge mode，从而通知 IE 采用其所支持的最新的绘制模式。
3.在html标签上最好带上lang属性指定当前网页的语言类型

<html lang="zh-CN">

</html>

这个表示使用当前的网页语言是简体中文,如果是英文的话,lang=”en”
4.

<link rel="stylesheet"> 
 <script></script>

这两个标签默认会带上自己的type类型,所以不用指定.link标签要带上rel属性表示他是一个css样式表
5.在书写html标签时不要在自闭合的标签后面加上斜杠.

<!doctype html>
<html lang="zh-CN">
   <head>
    <meta charset="UTF-8">
    <title>你最棒</title>
    <link rel="stylesheet" href="">
    <script src=""></script>
</head>
  <body>

  </body>
</html>

6.在标签缩进时最好用两个空格代替Tab建,因为Tab建进行缩进在不同的操作系统上的空格数量是不同的.
两个空格代替Tab建确保在每个系统上都是一致的
7.标签中的属性顺序

  1. class
  2. id
  3. data-set
  4. style href type value
  5. title alt
  6. role ,….

7.在标签中当是boolean类型属性时只写属性名就行了如:

<input type="checkbox" checked>
<input type="text" disabled>

下面来说说css的一些编码规范

在css中书写属性时遵循下面的顺序
1.positioning
2.Box model
3.Typographic
4.visual
5.Misc
如: 

.box {
  /*positioning*/
  position: absolute;
  top: 0;
  left: 0;
  bottom: 0;
  right: 0;
  z-index: 99;
  /*box model*/
  display: block;
  float: left;
  width: 200px;
  height: 200px;
  /*Typographic*/
  font-family: "Helvetica Neue", Ubuntu, Helvetica, Arial, sans-serif;
  font-size: 16px;
  color: #00aced;
  text-align: center;
  /*visual*/
  background-color: red;
  background-size: cover;
  border: 1px solid black;
  /*misc*/
  opacity: .8;
}

为什么要这样呢!position将使元素脱离标准流,会覆盖标准流中的元素,box model将会影响元素的大小,显示模式.
..__..
在写颜色属性值时使用十六进制的形式.后三位会前三位相同要省略如:
color: #edfced;
在写属性时最后一个属性的机尾要带上分号
属性连写时,为好区分每一个,分开写 

.box {
  border-bottom: 1px solid red;
  border-top: 1px solid red;
}

在css中一个选择器只包含一个属性时,将们写在一行

.box {border-bottom: 1px solid red;}
