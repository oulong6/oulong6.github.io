javascript 中的this

this在函数中的绑定有下列几种方式

1.隐式绑定


2. 通过对象调用函数this隐式绑定的方式

  1. 通过调用Function.prototype.call()方法显示绑定this call(this,params)方法的参数是this,params是多个参数.当this为null时this是指向window的
  2. 通过调用apply()方法显示绑定this apply(this,[]|arguments)的参数是this的绑定对象,与函数参数,可以是一个数组,也可以是一个类数组对象,比如arguments
  3. 通过bind()方法将this显示的绑定对象 bind(this)的参数是要绑定的this,方法会返回一个新的方法,这个方法的this是person

    this的一些理解

  4. 不要把this与作用域撤上关系 每当你想要把 this 和词法作用域的查找混合使用时，一定要提醒自己，这是无法实现的。 在字段代码中我们想要的是打印2,但是这是错误的为什么呢!你想一想当执行foo()时foo中的this是隐式的绑定window,
    当执行this.bar()时this window调用bar的所以bar中的this也是隐式的绑定window,但执行打印this.a时是在window上
    寻找a这个属性,当在window上找不到a时对返回undefined;所以会打印undefined;
    不要把作用域与this撤上关系!!!!

1.3 this到底是什么
排除了一些错误理解之后，我们来看看 this 到底是一种什么样的机制。
之前我们说过 this 是在运行时进行绑定的，并不是在编写时绑定，它的上下文取决于函数调
用时的各种条件。this 的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式。
当一个函数被调用时，会创建一个活动记录（有时候也称为执行上下文）。这个记录会包
含函数在哪里被调用（调用栈）、函数的调用方法、传入的参数等信息。this 就是记录的
其中一个属性，会在函数执行的过程中用到。
在下一章我们会学习如何寻找函数的调用位置，从而判断函数在执行过程中会如何绑定
this。
