js中的作用域

js常被称为动态,解释型语言,但是在你所不知的javascript一书中js其实是编译型语言
编译型语言通常包含以下几个步骤:
1.代码先进行分词/词法分析
2.解析/语法分析(编译器将词法分析的词法单元生成一颗抽象语法树)
3.编译器将这一课抽象语法树通过编译生成代码,也就是一系列的机械指令

编译器在编译代码时会与作用域相互作用

如编译器在看到下面这段代码实时:

function x() {
    var a = 23;
}

在函数x中看到var a时会让作用域分配一个变量标识符a的存储空间.

要执行代码时会有js引擎,作用域,编译器相互协作

如:

var a = 12;
function f(s) {
  console.log(a+s)
}
f(23);

在执行这段代码时.引擎会与作用域有下列的对话:
引擎:嗨作用域我要f(rsh)右查询
作用域:我有f,它是一个函数,geini
引擎:哦原来是全局作用域大哥啊,好呢!我这就执行它.
引擎: f函数里的作用域我要s标识符,
f中的作用域: 我有s标识符.它是一个形参,给你.
引擎:好呢!我将23赋值个形参s
引擎: f函数的作用域我要console.log
f中的引擎: 找到了它是一个内置函数给你.
引擎: 好呢!我执行它,诶!我需要a
f中的引擎: 我没见过.走开
引擎:哟!脾气好坏哦!我去你的三层作用域中找.
引擎: f函数的上层作用域我要a变量.0_0是全局大哥啊
全局作用域: 呵呵!我有给你
引擎:全局大哥真棒!哟!..我还要s变量(右查询)
f函数的作用域:你不记得了它是形参啊.哎给你
引擎:哟!我哪里不记得了,我只是想确认一下..嘻嘻
(4次右查询,1次左查询)

作用域是根据名称查询变量的一套规则!!!!

无论函数在哪里被调用,也无论它如何被调用,它的词法作用域都只由函数被声明时所处的位置决定!!!
